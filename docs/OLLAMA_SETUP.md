# Ollama Integration Guide: AI-Powered Risk Explanations

Using local Ollama models to generate intelligent, offline explanations for fraud risk components in LedgerSpy.

## Overview

The Ollama integration adds AI-powered explanations to the Risk Breakdown dashboard component. Instead of static text, each risk component (Anomaly Detection, Vendor Matching, Benford's Law) now gets dynamic explanations generated by a local LLM running on your machine.

**Key Benefits:**
- ✅ **Offline**: Runs completely locally, no cloud calls
- ✅ **Fast**: 6GB RAM models generate responses in 2-5 seconds
- ✅ **Explainable**: Auditors can understand *why* risk was flagged
- ✅ **Customizable**: Swap models easily (Mistral, Neural Chat, Llama, Orca)
- ✅ **Cost-Free**: No API fees, open-source models

## Setup Instructions

### Step 1: Install Ollama

Download from [https://ollama.ai](https://ollama.ai) (available for macOS, Linux, Windows)

**On Windows:**
1. Download `Ollama-Setup.exe`
2. Run installer, accept defaults
3. Restart computer
4. Ollama will auto-start in background

**On macOS:**
```bash
# Download DMG and drag to Applications
# Or use Homebrew:
brew install ollama
```

**On Linux:**
```bash
curl https://ollama.ai/install.sh | sh
```

### Step 2: Pull a Lightweight Model

Choose based on your RAM (6GB recommended):

```bash
# RECOMMENDED: Best balance (4.5GB)
ollama pull mistral

# Alternative: Faster for quick responses (4GB)
ollama pull neural-chat

# Alternative: Ultra-lightweight (2GB)
ollama pull orca-mini

# Alternative: General purpose (4GB)
ollama pull llama2
```

Check available models:
```bash
ollama list
```

### Step 3: Start Ollama Server

```bash
# Start Ollama server (runs on http://localhost:11434)
ollama serve
```

**Output should show:**
```
Loaded in 1.2ms
listening on 127.0.0.1:11434
```

Leave this terminal running (or run as background service).

### Step 4: Verify Ollama is Working

```bash
# In another terminal:
curl http://localhost:11434/api/tags
```

Should return:
```json
{
  "models": [
    {
      "name": "mistral:latest",
      "modified_at": "2024-04-19T10:30:00.000Z",
      "size": 4700000000,
      "digest": "..."
    }
  ]
}
```

### Step 5: Start LedgerSpy Backend

The backend will automatically detect Ollama and enable AI explanations.

```bash
cd backend
python run.py
```

Check `/api/risk/ollama-status` endpoint:
```bash
curl http://localhost:8000/api/risk/ollama-status
```

Response:
```json
{
  "status": "available",
  "ollama_url": "http://localhost:11434",
  "available": true,
  "models": ["mistral"],
  "recommended_model": "mistral",
  "message": "✓ Ollama is running"
}
```

### Step 6: Use in Frontend

The `ExplainableRiskBreakdown` component automatically:
1. Checks Ollama availability
2. Generates explanations for each risk component
3. Falls back gracefully if Ollama is offline

```jsx
import ExplainableRiskBreakdown from './components/ExplainableRiskBreakdown';

function Dashboard() {
  return (
    <ExplainableRiskBreakdown
      anomalyScore={45}
      vendorScore={30}
      benfordScore={20}
      useAIExplanations={true}
      ollamaBaseUrl="http://localhost:11434"
      ollamaModel="mistral"
    />
  );
}
```

## Troubleshooting

### "Ollama not available" message

**Problem:** Component shows ⚠️ Ollama Offline badge

**Solutions:**
1. **Check if Ollama is running:**
   ```bash
   curl http://localhost:11434/api/tags
   ```
   If no response, start Ollama: `ollama serve`

2. **Check if model is installed:**
   ```bash
   ollama list
   ```
   If empty, pull model: `ollama pull mistral`

3. **Check firewall:**
   - Port 11434 must be accessible
   - Run `netstat -an | find "11434"` on Windows

4. **Reset Ollama:**
   ```bash
   # Kill any existing processes
   pkill ollama
   
   # Clear cache and restart
   ollama serve --skip-load
   ```

### Slow Response Times

**Problem:** Explanations take 30+ seconds to generate

**Solutions:**
1. **Switch to faster model:**
   ```bash
   ollama pull neural-chat
   # Update component: ollamaModel="neural-chat"
   ```

2. **Check system resources:**
   - Look for swap usage (indicates insufficient RAM)
   - Monitor CPU/GPU usage during generation

3. **Reduce prompt complexity:**
   - Adjust prompts in `ollama_service.py` to be shorter
   - Lower `temperature` from 0.3 to 0.1 for faster responses

### Out of Memory (OOM) Errors

**Problem:** System freezes or Ollama crashes with OOM error

**Solutions:**
1. **Use smaller model:**
   ```bash
   ollama pull orca-mini  # 2GB instead of 4.5GB
   ```

2. **Increase system swap:**
   - Windows: Settings > System > Advanced system settings > Virtual memory
   - macOS: Already handles automatically
   - Linux: Create swap file

3. **Disable other applications:**
   - Close browser tabs, Discord, etc.
   - Free up 2GB+ of RAM

### High CPU Usage

**Problem:** Computer feels sluggish during explanation generation

**Solutions:**
1. **Use GPU acceleration (if available):**
   - NVIDIA GPU: Ollama auto-detects CUDA
   - AMD GPU: Set `OLLAMA_METAL=true`
   - Apple Silicon: Auto-enabled

2. **Reduce batch size:**
   - Modify prompts to ask for shorter responses
   - Edit `temperature` parameter in `_query_ollama()`

## Performance Benchmarks

Tested on typical 6GB RAM system with **mistral** model:

```

1. Single Component Explanation: 2-4 seconds

3. With Summary + Recommendations: 8-12 seconds

Memory Usage:
- Ollama Server Idle: ~100MB
- During Inference: ~4.5GB (peak)

Latency by Network:

- Local network (192.168.*): <500ms
- Remote network: 1-2s network overhead
```

## Advanced: Custom Prompts

To customize explanations, edit `ollama_service.py`:

```python
def generate_anomaly_explanation(self, anomaly_score, total_anomalies, flagged_transactions):
    # EDIT THIS PROMPT:
    prompt = f"""Provide a brief, technical explanation...
    
    Risk Score: {anomaly_score:.1f}%
    ...
    """
    return self._query_ollama(prompt)
```

**Tips:**
- Keep prompts concise (ollama processes faster with short prompts)
- Include specific metrics in prompt for better context
- Use low temperature (0.3) for consistent, focused responses
- Test changes with different anomaly/vendor/benford scores

## FAQ

**Q: Does Ollama phone home or send data to cloud?**
A: No. Ollama is completely local and offline. No data leaves your machine.

**Q: Can I use Ollama on Windows?**
A: Yes! Windows 10/11 with 6GB+ RAM required. Installs as service.

**Q: What if I restart my computer?**
A: Ollama auto-starts as service. Models are cached locally, no re-download needed.

**Q: Can I switch models without restarting?**
A: Yes. Just run `ollama pull <model>` and update `ollamaModel` prop.

**Q: What's the latency from browser to explanation?**
A: ~50ms to API + 2-5s for model inference = 2-5.5s total.

**Q: Does this work with AMD/Intel GPUs?**
A: Mistral, Neural Chat work on CPU. For GPU acceleration, check Ollama docs for your setup.

**Q: How much disk space do I need?**
A: ~6GB for model + cache. Ollama handles cleanup automatically.
